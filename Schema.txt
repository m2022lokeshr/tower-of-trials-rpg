SCHEMA — Tower of Trials RPG Combat Prototype
============================================

FILE: rpg_combat.html
TYPE: Single-file browser game (HTML + CSS + JS)


--- OVERVIEW ---

Turn-based text RPG. Player (Swordsman) climbs the Tower of Trials.
Page load opens Floor Selection after save data loads.
Player selects an unlocked floor, clicks "Ascend", then fights that floor's monsters in sequence.
After clearing a non-boss floor, player can Rest, Push Deeper, or Retreat.
Floors 1-10 are implemented. Floor 10 is a boss-only floor.
After floor 10, show "More floors coming soon".
Game state auto-saves to localStorage.

--- TOWER FLOW ---

Select:
  Floor Selection overlay is visible.
  Player chooses an unlocked floor.

Ready:
  Player can shop, manage equipment, retreat, or click "Ascend".

Combat:
  Player fights each monster in floor order.
  Current monster and floor queue persist in save data.
  Slime split must be cleared before the next queued monster appears.

Cleared:
  Player can Rest, Push Deeper, or Retreat.

Complete:
  After floor 10, show "More floors coming soon".
  Rest and Retreat remain available outside combat.

Retreat:
  Available outside active combat.
  Heals player to full HP.
  Resets current floor progress.
  Saves level, EXP, gold, bag contents, gear, and unlocks.
  Returns to Floor Selection.


--- EXP SYSTEM ---

Formula: EXP needed = 10 × (Level ^ 2.2)
Round to nearest integer.

On level up: stats increase, HP/MP fully restored, overflow EXP carries over.


--- COMBAT TURN ORDER ---

1. Player picks action → resolves immediately
2. If monster alive → monster attacks
3. If monster dies → award EXP/gold, try drops, level up, handle gimmick, advance queue
4. Cooldowns tick down at end of player turn (only on non-Power-Strike turns)

Buttons disabled during resolution to prevent double-clicks.


--- DAMAGE FORMULAS ---

Player damage = (Total ATK × multiplier) − (Monster DEF / 2)
Minimum damage = 1

Total ATK = base ATK + all equipment ATK bonuses

Multipliers:
- Attack:       ×1.0
- Slash:        ×1.0 (crit chance → ×1.5)
- Power Strike: ×1.5 (unlock Lv 10, cooldown 3 turns)

Monster damage = Monster ATK − (Total Player DEF / 2)
Minimum damage = 1

Total Player DEF = base DEF + all equipment DEF bonuses

Guard: reduces incoming damage by 30% (×0.7), min 1.

Crit chance = 10% base + equipment crit bonuses.

--- HEALTH MANAGEMENT ---

Player starts with 1 Health Potion in bag.

Health Potion:
  Restores 30 HP, up to max HP
  Usable in combat from the action bar
  Consumes the player's turn, then monster attacks
  Hidden when potion count is 0
  Disabled at full HP

Potion drops:
  10% chance on defeated non-boss, non-mini monsters
  Mini-Slime does not drop potions

Rest:
  Available after a floor is cleared
  Restores HP to full
  No EXP or gold cost


--- BAG SYSTEM ---

Max slots: 5
Max potion stack: 10 per slot

Bag holds any mix of:
  - Health Potions (stackable up to 10)
  - Gear items (1 per slot, non-stackable)

Bag UI:
  Shows 5 slots with contents
  Potions: show count, sell button, drop button
  Gear: show name, stat text, equip button, sell button, drop button
  Equipped section below shows current gear per slot with unequip button
  "Sell All" button sells entire bag for gold

Bag functions:
  addPotionToBag()  — stacks if possible, else fills empty slot
  addGearToBag()    — fills first empty slot
  removeFirstPotion() — removes one potion from first stack found
  bagIsFull()       — true if no empty slots

--- COOLDOWN SYSTEM ---

Power Strike: 3 turn cooldown.
After use, cannot use for next 3 player turns.
Cooldown decrements at end of each NON-Power-Strike player turn.
Button shows remaining turns when disabled.


--- DEATH / RESTART ---

Player HP floor: cannot die in 1 hit unless already at 1 HP.
  (If HP > 1 and damage would bring it to 0 or below, set HP to 1.)

On death:
  Lose 10% current gold, rounded down
  Restore full HP
  Show "You Died" overlay with animation
  Display gold lost
  Two buttons: "Return to Camp" (keeps progress, returns to Camp)
               "New Game" (full reset)
  Retain level, EXP, equipment, bag, and unlocks

Restart (New Game):
  Resets everything to Lv 1, floor 1, starter potion, no gear, no gold, clears log.
  Overwrites saved game with fresh state.


--- SAVE / LOAD ---

Storage: localStorage
Key: swordsmanRpgSave
Version: 4

Auto-save triggers:
  New game initialization
  End of player turn
  Ascend, Rest, Push Deeper, Retreat
  Shop purchases
  Equip / Unequip / Drop / Sell
  Death penalty
  Restart

Saved player state:
  Level, EXP, base stats, HP/max HP, MP, gold
  Power Strike cooldown, guard active flag
  Equipment (5 slot keys)
  Bag (5 slots with type + count/key)

Saved combat state:
  Current monster template key, name, rank, HP/max HP, ATK, DEF
  EXP reward, gold reward, flags (splits, isMini, isBoss)

Saved tower state:
  Current floor
  Phase: select / ready / combat / cleared / complete
  Current floor plan (monster key array)
  Current monster index
  Highest floor reached
  Unlocked floors array
  More-floors-coming flag

Load behavior:
  On page open, load save first
  Then show the Camp dashboard
  Older v3 saves are migrated to v4 bag format
  v2 saves are rejected


--- WEB AUDIO SOUND ENGINE ---

Uses Web Audio API (AudioContext) with oscillator-based sounds.
No external audio files.

Sound IDs:
  attack  → 220 Hz square wave, short beep
  hit     → 120 Hz sawtooth, thud
  crit    → 330 Hz + 440 Hz square wave, two-tone
  levelup → 523 Hz + 659 Hz + 784 Hz sine wave, ascending chime
  die     → 180 Hz + 100 Hz sawtooth, descending tone

AudioContext resumes on first sound play if suspended.
